셀 병합 | Cell Merging

FlexGrid는 내용(컨텐츠) 중심의 셀 병합을 지원합니다.

셀 병합을 사용하려면 allowMerging 속성을 설정하여 병합할 그리드의 일부 또는 부분을 지정하고, 특정 행과 열의 allowMerging 속성을 true로 설정하십시오. 그렇게하면 그리드가 동일한 내용의 셀을 병합하여 데이터를 시각적으로 그룹화합니다.

Members

Name Value Description
None 0

병합 안함.
Cells 1

스크롤 가능한 셀 병합.
ColumnHeaders 2

열(컬럼) 헤더 병합.
RowHeaders 4

행(로우) 헤더 병합.
AllHeaders ColumnHeaders | RowHeaders

열/행 헤더 병합.
All Cells | AllHeaders

헤더 및 셀 모두 벙합.

아래 예제에서 allowMerging 속성은 "Country"및 "Active"열에서 true로 설정됩니다. 

// 행렬 헤더의 크기 변경을 금지합니다.
var theGrid = new wijmo.grid.FlexGrid('#theGrid', {
    allowMerging: 'Cells',    
    columns: [
      { binding: 'country', header: 'Country', allowMerging: true  },
      { binding: 'sales', header: 'Sales', format: 'n2' },
      { binding: 'expenses', header: 'Expenses', format: 'n2' },
      { binding: 'active', header: 'Active',  allowMerging: true },
    ],
    itemsSource: new wijmo.collections.CollectionView(data, {
    	sortDescriptions: ['country', 'active']
    })
});

사용자 정의 병합 | Custom Merging


기본적으로 FlexGrid는 내용을 기반으로 셀에 병합을 적용합니다. 경우에 따라 병합에 다양한 전략을 사용하는 것이 좋습니다.
wijmo.grid.MergeManager 클래스를 확장하는 클래스를 정의하고 해당 유형의 객체를 그리드의 mergeManager 속성에 할당하면됩니다.

해당 샘플을 보시면, 저의 쪽에서 미리 사용자 정의 "MergeManager" 클래스를 구현해 놓았습니다.
사용자 지정 병합 관리자(Merge Manager)가 행과 열을 모두 포함하는 병합 된 범위를 만드는 방법에 유의하십시오. 이것은 기본 병합 관리자가 수행하지 않는 작업입니다.

이를 그대로 프로젝트에 js 파일 또는 스크립트에 붙혀넣어 사용하시면, 행/열에 관계 없이 동일한 데이터에 대해서 자동으로 병합을 수행합니다.

제한된 병합 | Restricted Merging


이미 보신 것과 같이 기본적으로 FlexGrid는 인접 셀의 내용에 관계없이 동일한 내용을 가진 셀을 병합합니다.
하자만, 위의 사용자 정의 클래스를 변형하여, 인접한 셀(열/column)들의 내용이 같을 경우에만 동일한 데이터로서 병합을 할 수 있도록 구현할 수 있습니다.

해당 샘플을 보시면, 저의 쪽에서 미리 사용자 정의 "RestrictedMergeManager" 클래스를 구현해 놓았습니다.
이를 사용하여, 이전 열의 셀에 같은 값이 들어있는 경우에만 셀을 세로로 병합하는 사용자 지정 병합 관리자를 사용합니다.

이를 그대로 프로젝트에 js 파일 또는 스크립트에 붙혀넣어 사용하시면, 행/열에 관계 없이 동일한 데이터에 대해서 자동으로 병합을 수행합니다.
