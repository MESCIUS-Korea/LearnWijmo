딥 바인딩 | Deep Binding

각 그리드 열에는 셀에 표시되어야하는 데이터 항목의 속성을 결정하는 바인딩 속성이 있습니다. 대부분의 경우이 속성은 'name'또는 'value'와 같은 일반 식별자입니다.

그러나 그리드는 또한 '딥 바인딩 (deep-binding)'을 지원합니다. 여기서 바인딩 문자열은 마침표로 구분 된 식별자로 구성된 바인딩 경로를 포함합니다.
구문은 일반 JavaScript 표현식에서 사용하는 것과 유사합니다.

예를 들어 itemsSource 배열의 데이터 객체에 'Customer'객체에 대한 참조가 포함 된 'customer'속성이 있으면 'customer.name' 바인딩을 사용하여 표에 고객의 이름을 표시 할 수 있습니다.

Result와 함게 HTML, JS 코드도 확인이 가능합니다.

Deep binding은 여러 시나리오에서 매우 유용 할 수 있지만 바인딩이 작동하는 방식을 알고 있어야합니다. 이 예에서 고객의 성과 이름을 편집하면 동일한 고객에게 바인드 된 모든 행에 영향을줍니다.

또한 일반 바인딩과 달리 딥 바인딩은 기본적으로 '실행 취소(Undo)'를 할 수 없습니다. 딥 바인딩에 대해 실행 취소를 사용하려면 그리드의 rowEditStarted 이벤트를 사용하여 원래의 딥(Deep) 바운드 값을 기록하고, 편집이 취소될 경우 해당 값을 복원하는 rowEditEnding 이벤트를 사용해야합니다.
