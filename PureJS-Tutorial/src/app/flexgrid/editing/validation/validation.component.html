유효성 검사 | Validation

FlexGrid는 여러 가지 방법으로 유효성 검사를 도와줍니다:

  1. Automatic Type Coercion: 셀 편집이 커밋되면 그리드는 열의 데이터 유형과 일치하도록 값을 자동으로 강제 변환합니다. 또한 열의 isRequired 속성을 기반으로 null 및 빈(empty) 값을 확인합니다. 데이터가 유효하지 않으면 원래 셀 값이 복원됩니다.
  2. Data Maps: 그리드에 특정 입력 값만 허용해야하는 열이있는 경우, 사용자가 셀에 잘못된 내용을 입력 할 수 없도록 열에 DataMap을 추가하십시오.
  3. Event-Based Validation: 그리드는 편집기의 현재 값을 확인하고 새 값이 유효하지 않은 경우, 편집을 취소 할 수있는 cellEditEnding 이벤트를 발생시킵니다.
  4. CollectionView-Based Validation: 그리드는 콜렉션 뷰의 getError 메소드를 사용하여 잘못된 셀을 강조 표시하고 사용자가 잘못된 정보를 입력하는 것을 방지합니다.
