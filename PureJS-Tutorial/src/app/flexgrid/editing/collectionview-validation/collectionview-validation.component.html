컬렉션뷰 유효성 검사 | CollectionView Validation

FlexGrid는 Collection View 클래스와 함께 작동하여 항목 및 컬렉션 수준의 유효성 검사를 제공합니다.

이 기능을 사용하려면 유효성을 검사할 데이터 항목과 유효성을 검사할 속성을 포함하는 두 개의 매개변수를 취하는 함수에 getError를 설정하고 오류 조건을 설명하는 문자열을 반환합니다 (오류가없는 경우 null).

아래의 그리드에는 데이터 소스로 'Sales'및 'Expenses'속성에 양수 값이 포함되도록하는 getError 함수가 구현되어 있습니다. 유효성 검사가 어떻게 작동하는지 보려면 두 열 중 하나에 음수 값을 입력하십시오.

샘플코드:

Result와 함게 HTML, JS 코드도 확인이 가능합니다.

그리드에는 유효성 검사 동작을 사용자가 지정할 수있는 두 가지 속성이 있습니다.:

  • showErrors: 격자가 유효성 검사 오류가 포함 된 셀과 오류 설명이 포함 된 툴팁에 'wj-state-invalid'클래스를 추가해야하는지 여부.
  • validateEdits: 사용자가 유효성 검사를 실패한 편집을 커밋하려고 할 때 그리드가 편집 모드로 유지되어야하는지 여부.
